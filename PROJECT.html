NGP Dashboard
Documentação do projeto
Plataforma de gestão de campanhas Meta Ads para a agência NGP e seus clientes. Dados em tempo real, relatórios automatizados e controle centralizado.
Next.js 16 Supabase TypeScript Meta Ads API Chart.js Deno Edge Functions

Visão Geral

📊
Dashboard NGP (Admin)
Visão completa de todas as contas de clientes. Seleção de conta, KPIs, gráficos, campanhas, relatórios salvos e plataformas integradas.
👤
Dashboard Cliente
Acesso restrito ao próprio account_id. Exibe métricas de campanha, investimento, leads, conversas, ROAS e CTR em tempo real.
📄
Gerador de Relatórios
Relatório semanal com auto-importação da Meta API. Salva na nuvem (Supabase), compartilhável via link, exportável em PDF.
🔗
UTM Builder & IA
Gerador de parâmetros UTM para rastreamento de campanhas e painel de análise por inteligência artificial.

Arquitetura

O projeto usa uma arquitetura híbrida com duas camadas de frontend coexistindo.

Fluxo de autenticação

Login (HTML ou Next.js)
→
Token salvo no Supabase
→
SessionStorage
→
Role check
→
Dashboard correto

Camadas do sistema

Camada 1
Next.js (app/)
Autenticação, roteamento, dashboard cliente React, gerador de relatórios.
Camada 2
HTML Estático (public/)
Dashboard NGP completo, relatório Pro, UTM Builder, IA — toda lógica no browser com Chart.js.
Camada 3
Supabase Backend
PostgreSQL, Edge Functions (Deno), Storage para avatares, autenticação customizada.

Roles de Usuário

Administrador
NGP Admin
  • Visualiza e gerencia todos os clientes
  • Cria, edita e configura contas de clientes
  • Acessa métricas de qualquer Meta account
  • Cria e exclui relatórios de qualquer cliente
  • Upload de fotos de perfil dos clientes
  • Acesso ao dashboard completo (adsboard)
Cliente
Cliente NGP
  • Vê apenas o próprio account Meta Ads
  • Acessa métricas de campanhas em tempo real
  • Visualiza relatórios criados para si
  • Pode excluir apenas os próprios relatórios
  • Edita próprio perfil e avatar
  • Sem acesso ao painel de gestão NGP

Edge Functions (Supabase / Deno)

login
Autentica usuário, cria token de sessão na tabela sessions com expiração.
logout
Invalida o token de sessão. Chamado ao clicar em "Sair".
meta-proxy
Proxy para a Meta Ads API. O token Meta nunca é exposto ao browser — fica somente no backend.
get-ngp-data
Retorna lista de clientes (id, nome, username, meta_account_id, foto_url) para o seletor de contas. Exclusivo NGP.
add-cliente
Cria novo cliente no banco (nome, username, password_hash, meta_account_id, ativo). Valida duplicidade de username.
update-cliente
Atualiza dados do cliente (nome, meta_account_id) e faz upload de avatar no bucket avatars com cache-busting.
save-relatorio
Cria ou atualiza relatório (INSERT/UPDATE). Salva cliente_id e cliente_username. Validação de ownership para clientes.
delete-relatorio
Exclui relatório por UUID. NGP pode excluir qualquer um; cliente só exclui os próprios.

Banco de Dados

usuarios
id (uuid PK) nome (text) username (text UNIQUE) password_hash (text) role (text) ativo (bool default true) meta_account_id (text) meta_token (text) foto_url (text) created_at (timestamptz)
sessions
id (uuid PK) token (text UNIQUE) usuario_id (uuid FK) expires_at (timestamptz) created_at (timestamptz)
relatorios
id (uuid PK) cliente_id (uuid FK) cliente_username (text) titulo (text) periodo (text) dados (jsonb) publicado (bool) created_by (uuid) created_at (timestamptz) updated_at (timestamptz)
avatars (Storage bucket)
Bucket público Path: {username}/avatar.{ext} Tipos: jpg, png, webp Limite: 2MB

Instalação & Execução

# Instalar dependências
npm install

# Rodar em desenvolvimento
npm run dev

# Build para produção
npm run build

# Deploy das Edge Functions (Supabase CLI)
supabase functions deploy add-cliente --no-verify-jwt
supabase functions deploy update-cliente --no-verify-jwt
supabase functions deploy delete-relatorio --no-verify-jwt
supabase functions deploy get-ngp-data --no-verify-jwt
supabase functions deploy save-relatorio --no-verify-jwt
